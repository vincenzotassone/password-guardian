Divisione del lavoro nel progetto Python

Introduzione

Durante lo sviluppo del progetto Python abbiamo lavorato entrambi insieme alla realizzazione dell’applicazione. La collaborazione è avvenuta utilizzando lo stesso computer e lo stesso ambiente di sviluppo, quindi anche lo stesso account GitHub configurato sul dispositivo.

Per questo motivo, analizzando la cronologia dei commit Git, tutti i commit risultano associati allo stesso account GitHub. Questo non significa che il lavoro sia stato svolto da una sola persona, ma semplicemente che abbiamo utilizzato lo stesso profilo Git per caricare le modifiche sul repository.

Modalità di collaborazione

Abbiamo preferito lavorare insieme sullo stesso computer in modo da poter confrontare le idee, correggere gli errori e prendere decisioni condivise durante lo sviluppo.

Ogni parte del progetto è stata discussa e realizzata insieme, con entrambi coinvolti nelle varie fasi:

* progettazione della struttura del programma;
* scrittura e modifica del codice Python;
* creazione delle classi e delle funzionalità;
* test dell’applicazione;
* creazione della documentazione;
* gestione del repository GitHub.

Divisione delle attività

Anche se abbiamo lavorato insieme, le attività principali sono state organizzate nel seguente modo:

Sviluppo del codice Python

Abbiamo collaborato alla realizzazione delle principali funzionalità del progetto:

* creazione del generatore di password sicure;
* sviluppo dell’analizzatore delle password;
* gestione delle policy configurabili;
* calcolo dell’entropia e del livello di sicurezza;
* gestione dei report JSON;
* organizzazione del codice tramite programmazione orientata agli oggetti (OOP).

Struttura del progetto e documentazione

Abbiamo lavorato insieme anche alla parte di organizzazione del progetto:

* creazione della struttura delle cartelle (src, tests, docs);
* scrittura dei manuali utente e tecnico;
* descrizione delle scelte progettuali;
* creazione del devlog;
* aggiornamento del README.

Test e verifica

Abbiamo entrambi partecipato alla fase di verifica del programma:

* esecuzione dei test;
* ricerca e correzione degli errori;
* controllo del corretto funzionamento delle funzionalità sviluppate.

Gestione dei commit GitHub

Tutti i commit presenti nel repository risultano provenire dallo stesso account GitHub perché abbiamo utilizzato lo stesso computer e lo stesso account Git configurato.

La cronologia dei commit rappresenta quindi le modifiche effettuate sul progetto, ma non permette di distinguere quale dei due membri del gruppo abbia materialmente scritto ogni singola modifica.

La realizzazione del progetto è stata comunque un’attività condivisa, dove entrambi abbiamo partecipato allo sviluppo e alle decisioni tecniche.

Conclusione

Il progetto è stato sviluppato attraverso una collaborazione continua tra i due membri del gruppo. La scelta di lavorare sullo stesso dispositivo ci ha permesso di lavorare più facilmente insieme, anche se ha comportato che GitHub mostrasse tutti i commit associati allo stesso account.

Entrambi abbiamo contribuito alla progettazione, allo sviluppo del codice e alla documentazione finale del progetto.
